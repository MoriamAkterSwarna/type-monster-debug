Local Storage: This read-only interface property provides access to the Document’s local storage object, the stored data is stored across browser sessions. Similar to sessionStorage, except that localStorage data gets cleared when the page session ends – that is when the page is closed. It is cleared when the last “private” tab of a browser is closed.
Local storage has 4 methods:

  • setItem() Method: localStorage.setItem(key, value)
  • getItem() Method: localStorage.getItem(key)
  • removeItem() Method: localStorage.removeItem(key)
  • clear() Method: localStorage.clear()

Session Storage: Session Storage objects can be accessed using the sessionStorage read-only property. The difference between sessionStorage and localStorage is that localStorage data does not expire, whereas sessionStorage data is cleared when the page session ends.
Session Storage has 4 methods:
  • setItem() Method: sessionStorage.setItem(key, value)
  • getIteam() Method: sessionStorage.getItem(key)
  • removeItem() Method: sessionStorage.removeItem(key)
  • clear() Method: sessionStorage.clear()

Global scope: Global scope contains all of the things defined outside of all code blocks. A code block simply consists of grouped statements inside curly braces ({ }). if statement, loops, function are examples of structure that create a code block. A global variable has global scope. A global variable is accessible from anywhere in the code.

Local Scope:Local scope contains things defined inside code blocks. A local variable has local scope. A local variable is only accessible where it’s declared. If local variable and global variable have same name then changing value of one variable does not affect on the value of another variable.

Event Loop:The event loop is the secret behind JavaScript’s asynchronous programming. JS executes all operations on a single thread, but using a few smart data structures, it gives us the illusion of multi-threading. Let’s take a look at what happens on the back-end.
The call stack is responsible for keeping track of all the operations in line to be executed. Whenever a function is finished, it is popped from the stack.
The event loop facilitates this process; it constantly checks whether or not the call stack is empty. If it is empty, new functions are added from the event queue. If it is not, then the current function call is processed.

Undefined: From 6 primitive types undefined is a special value with its own type Undefined. Undefined value primitive value is used when a variable has not been assigned a value. Undefined can be created in some ways:
  1. Uninitialized variable: A declared variable but not yet assigned with a value (uninitialized) is by default undefined.
    Example: let number;
    number;
  2. Accessing a non-existing property: When accessing a non-existing object property, JavaScript returns undefined.
  3. Function parameters: Usually a function defined with a specific number of parameters should be invoked with the same number of arguments.
  4. Function return value: Implicitly, without return statement, a JavaScript function returns undefined..
  5. void operator: void evaluates the expression and returns undefined no matter the result of the evaluation.
  6. undefined in arrays: You get undefined when accessing an array element with an out of bounds index.
